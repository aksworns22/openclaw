LangChain은 같은 루프를 어떻게 감싸는가

Lesson 0002 · Lesson 0001에서 본 4단계가 LangChain에선 어떤 이름으로 바뀌는지만 본다

지난 레슨에서 본 4단계 루프(tool 선언 → tool_calls 요청 → 직접 실행 → 결과 반환)는 그대로다. 이번 레슨은 새 개념이 아니라 같은 4단계의 LangChain식 이름표를 붙이는 게 전부다.

이름만 바뀐다

단계raw ollama (Lesson 0001)LangChain
1. tool 선언tools=[get_temperature]@tool 데코레이터 + .bind_tools([...])
2. 모델의 요청response.message.tool_callsAIMessage.tool_calls
3. 직접 실행get_temperature(**call.function.arguments)get_temperature.invoke(tool_call['args'])
4. 결과 반환{"role": "tool", ...}ToolMessage(tool_call_id=..., content=...)

출처: LangChain Docs: ChatOllama integration

tool_calls 모양이 살짝 다르다

raw ollama에서는 call.function.name, call.function.arguments처럼 객체 속성으로 접근했다 (Lesson 0001 복습). LangChain의 AIMessage.tool_calls는 dict의 리스트다 — 그래서 []로 꺼낸다. 

# LangChain 쪽 tool_calls 구조
result.tool_calls
# [{'name': 'get_temperature', 'args': {'city': 'Seoul'}, 'id': '...', 'type': 'tool_call'}]

tool_call = result.tool_calls[0]
tool_call['name']   # 'get_temperature'   ← dict 접근, [] 사용
tool_call['args']   # {'city': 'Seoul'}
헷갈리기 쉬운 지점 — Lesson 0001에서 배운 "객체는 ., dict는 []" 규칙 그대로다. raw ollama의 ToolCall은 객체였지만, LangChain의 tool_call은 dict라서 .name이 아니라 ['name']이다.

실습: gemma4:e4b + ChatOllama.bind_tools

from typing import List
from langchain.messages import AIMessage, ToolMessage
from langchain.tools import tool
from langchain_ollama import ChatOllama

@tool
def get_temperature(city: str) -> str:
    """주어진 도시의 현재 기온을 반환한다.

    Args:
        city: 영어 도시 이름 (예: "Seoul", "Tokyo")
    """
    temperatures = {"Seoul": "26°C", "Tokyo": "24°C"}
    return temperatures.get(city, "데이터 없음")

llm = ChatOllama(model="gemma4:e4b", temperature=0).bind_tools([get_temperature])

# 1~2단계: tool 선언 + 모델의 요청
result = llm.invoke("서울 기온이 어때?")

if isinstance(result, AIMessage) and result.tool_calls:
    tool_call = result.tool_calls[0]
    print("모델이 요청한 함수:", tool_call['name'], tool_call['args'])

    # 3단계: 직접 실행
    tool_result = get_temperature.invoke(tool_call['args'])

    # 4단계: ToolMessage로 결과 반환 후 재호출
    messages = [result, ToolMessage(tool_call_id=tool_call['id'], content=str(tool_result))]
    final = llm.invoke(messages)
    print(final.content)

Lesson 0001과 똑같이, 한글 도시 이름을 물어보면 모델이 인자를 어떻게 채우는지(영어로 변환하는지, 한글 그대로 보내는지) 다시 한 번 확인해보자 — docstring의 Args: city: 영어 도시 이름 한 줄이 그 동작을 바꾸는지가 핵심 관찰 포인트다.
