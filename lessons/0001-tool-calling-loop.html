Tool Calling 루프의 정체

Lesson 0001 · 모델이 "글을 쓰는 것"에서 "행동을 요청하는 것"으로 넘어가는 순간을 본다

지금까지 ChatOllama로 모델을 불러서 텍스트나 JSON을 받아왔다. 이번 레슨에서 다루는 것은 딱 하나다 — 모델은 절대 함수를 직접 실행하지 않는다는 것, 그리고 그 대신 무슨 일이 일어나는지.

모델은 코드를 실행할 수 없다

Gemma 4 같은 모델은 다음에 올 토큰을 예측할 뿐이다. "날씨를 알려줘"라는 요청을 받으면, 모델은 실제로 날씨 API를 호출하는 게 아니라 "날씨 API를 이렇게 호출해야 한다"는 텍스트를 생성한다. 이 텍스트를 파싱해서 실제로 함수를 실행하는 건 항상 너의 코드다.

핵심 제약 — Google 공식 문서: "A Gemma model cannot execute code on its own." 모델이 만든 함수 호출 요청을 파싱하고 실행하는 책임은 전부 개발자에게 있다. — ai.google.dev: Function calling with Gemma 4

4단계 루프

이 제약을 받아들이면, tool calling은 다음 4단계의 반복일 뿐이라는 게 보인다.

1. Tool을 선언한다
사용 가능한 함수의 이름, 설명, 파라미터 스키마를 모델에게 알려준다. (JSON Schema 또는 Python 함수 + 타입 힌트)
2. 모델이 호출을 "요청"한다
모델은 직접 실행하지 않고, tool_calls라는 구조화된 데이터를 응답에 담아 돌려준다. Gemma 4는 내부적으로 <|tool_call|> 같은 특수 토큰으로 이걸 표현하지만, Ollama API를 쓰면 이미 파싱된 JSON으로 받는다.
3. 너의 코드가 실제로 실행한다
tool_calls 안의 함수 이름과 인자를 꺼내서, 너의 Python 코드가 그 함수를 직접 호출한다.
4. 결과를 모델에게 돌려준다
실행 결과를 role: "tool" 메시지로 대화 기록에 추가하고, 다시 모델을 호출한다. 모델은 이제 그 결과를 바탕으로 사람이 읽을 자연어 답을 생성한다.

출처: Ollama Docs: Tool calling · ollama/ollama api.md

실습: gemma4:e4b로 직접 돌려보기

네 머신에 이미 gemma4:e4b가 받아져 있고, tools 능력을 지원하는 걸 확인했다. 터미널에서:

pip install ollama

그리고 다음 스크립트를 실행해보자.

from ollama import chat

def get_temperature(city: str) -> str:
    """주어진 도시의 현재 기온을 반환한다"""
    temperatures = {"Seoul": "26°C", "Tokyo": "24°C"}
    return temperatures.get(city, "데이터 없음")

messages = [{"role": "user", "content": "서울 기온이 어때?"}]

# 1단계: tool 선언 (함수를 그대로 넘기면 SDK가 스키마를 자동 생성)
response = chat(model="gemma4:e4b", messages=messages, tools=[get_temperature])

# 2단계: 모델이 tool_calls로 "요청"했는지 확인
if response.message.tool_calls:
    call = response.message.tool_calls[0]
    print("모델이 요청한 함수:", call.function.name, call.function.arguments)

    # 3단계: 너의 코드가 직접 실행
    result = get_temperature(**call.function.arguments)

    # 4단계: 결과를 role:tool로 돌려주고 다시 호출
    messages.append(response.message)
    messages.append({"role": "tool", "tool_name": call.function.name, "content": str(result)})
    final = chat(model="gemma4:e4b", messages=messages, tools=[get_temperature])
    print(final.message.content)

모델이 tool_calls를 채워서 돌아오는지, 그리고 두 번째 호출에서 get_temperature의 결과를 반영한 자연어 문장을 만드는지 직접 확인해보자.
