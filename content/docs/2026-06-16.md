## LLM 지도: 딥러닝과 언어 모델링


### 핵심 개념


<p><strong>임베딩</strong>: 데이터의 의미와 특징을 포착해 숫자로 표현한 것</p>

<p><strong>언어 모델링</strong>: 모델이 입력받은 텍스트의 다음 단어를 예측해 텍스트를 생성하는 방식</p>
<ul>
  <li>언어 모델(language model): 다음 단어를 예측하는 방식으로 훈련한 모델</li>
</ul>

### 딥러닝 vs. 머신러닝


<p>딥러닝이 머신러닝과 가장 큰 차이를 보이는 지점: <strong>특징(feature)을 누가 뽑는가</strong></p>

<table>
  <tr><th></th><th>특징 추출 주체</th></tr>
  <tr><td>기존 머신러닝</td><td>연구자 / 개발자</td></tr>
  <tr><td>딥러닝</td><td>모델이 스스로</td></tr>
</table>

### LLM 역사의 세 가지 전환점


<table>
  <tr><th>연도</th><th>사건</th></tr>
  <tr><td>2013</td><td>word2vec — 단어를 벡터로 표현, 의미 유사도 포착</td></tr>
  <tr><td>2017</td><td>Transformer architecture 등장</td></tr>
  <tr><td>2018</td><td>GPT-1 — 언어 모델링으로 사전 학습 후 fine-tuning이 효과적임을 증명</td></tr>
</table>

### 전이 학습 (Transfer Learning)


<p>하나의 문제를 해결하는 과정에서 얻은 지식을 다른 문제에 활용하는 방식</p>

<ul>
  <li><strong>사전 학습(pre-training)</strong>: 대량의 데이터로 모델을 먼저 학습</li>
  <li><strong>미세 조정(fine-tuning)</strong>: 특정 다운스트림 과제를 위해 추가 학습</li>
</ul>

<p>NLP에서 돌파구: 2018년, 다음 단어를 예측하는 언어 모델링 방식으로 사전 학습 시 적은 레이블 데이터로도 기존 지도 학습을 뛰어넘는다는 것을 발견 → GPT-1</p>

### RNN의 한계


<p><strong>시퀀스</strong>: 작은 단위(단어)의 데이터가 연결되고, 길이가 다양한 데이터 형태</p>

<p>RNN은 하나의 잠재 상태(hidden state)에 지금까지의 입력 텍스트 맥락을 압축한다.</p>
<ul>
  <li>효율적이지만, 입력이 길어질수록 먼저 입력한 단어의 의미가 희석됨</li>
  <li>→ 이 한계를 극복하기 위해 Transformer(Attention 메커니즘)가 등장</li>
</ul>

<p>시퀀스가 길어질수록 초기 입력의 정보가 hidden state 안에서 점점 옅어지는 모습:</p>

<div class="mermaid">
graph LR
  w1["word 1"] --> h1["hidden 1<br>(word1 100%)"]
  h1 --> h2["hidden 2<br>(word1 70%)"]
  w2["word 2"] --> h2
  h2 --> h3["hidden 3<br>(word1 40%)"]
  w3["word 3"] --> h3
  h3 --> h4["hidden N<br>(word1 거의 소실)"]
  w4["..."] --> h4
</div>

### 언어 모델이 ChatGPT가 되기까지


<p>핵심 개념: <strong>트랜스포머 + 정렬(alignment)</strong></p>

<ul>
  <li>Transformer: 시퀀스 전체를 병렬로 처리, 장거리 의존성 포착 가능</li>
  <li>Alignment(정렬): 모델 출력이 인간의 의도에 맞도록 조정하는 과정 (RLHF 등)</li>
</ul>

## PyTorch로 구현하는 CharRNN: 이름으로 국적 분류


### 문제 정의


<p>이름(문자열) → 국적(18개 클래스) 분류 문제</p>
<ul>
  <li>데이터: 20,074개 이름, 18개 언어 (Czech, Arabic, Chinese, ...)</li>
  <li>입력: 문자 시퀀스 / 출력: 국적 레이블</li>
</ul>

### 데이터 전처리


#### 문자 인코딩


<p>유니코드 이름을 ASCII로 정규화한 뒤 one-hot 벡터로 변환</p>

<pre><code>allowed_chars = string.ascii_letters + " .,;'" + "_"  # 58개 문자
n_letters = len(allowed_chars)  # 58

def lineToTensor(line):
    tensor = torch.zeros(len(line), 1, n_letters)  # (seq_len, 1, 58)
    for li, letter in enumerate(line):
        tensor[li][0][letterToIndex(letter)] = 1
    return tensor
</code></pre>

<p><code>lineToTensor('Jones')</code> → shape <code>[5, 1, 58]</code>: 5글자 × 1배치 × 58차원 one-hot</p>

### 모델 구조: CharRNN


<p>이름 한 글자씩을 순서대로 RNN에 흘려보내고, 마지막 hidden state만 분류기로 넘기는 흐름:</p>

<div class="mermaid">
graph LR
  input["이름 (글자 시퀀스)"] --> onehot["one-hot 인코딩<br>(seq_len, 1, 58)"]
  onehot --> rnn["nn.RNN<br>hidden_size=128"]
  rnn --> hidden["마지막 hidden state"]
  hidden --> linear["nn.Linear<br>(128 → 18)"]
  linear --> softmax["LogSoftmax"]
  softmax --> output["국적 예측 (18개 중 1개)"]
</div>

<pre><code>class CharRNN(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        self.rnn = nn.RNN(input_size, hidden_size)   # 시퀀스 처리
        self.h2o = nn.Linear(hidden_size, output_size) # 최종 분류
        self.softmax = nn.LogSoftmax(dim=1)

    def forward(self, line_tensor):
        rnn_out, hidden = self.rnn(line_tensor)
        output = self.h2o(hidden[0])  # 마지막 hidden state만 사용
        return self.softmax(output)
</code></pre>

<p>하이퍼파라미터: <code>input_size=58</code>, <code>hidden_size=128</code>, <code>output_size=18</code></p>

### 핵심 포인트


<ul>
  <li>RNN은 시퀀스를 순서대로 처리하고, <strong>마지막 hidden state</strong>가 전체 시퀀스의 요약</li>
  <li>one-hot 인코딩은 단순하지만 어휘 크기가 작은 문자 수준 모델에서 효과적</li>
  <li><code>nn.RNN</code>은 <code>(seq_len, batch, input_size)</code> 형태의 입력을 받음</li>
</ul>

### RNN의 구조적 한계 (이론과 연결)


<p>이 모델은 하나의 hidden state에 전체 이름의 맥락을 압축한다. 이름처럼 짧은 시퀀스에서는 잘 동작하지만, 긴 텍스트에서는 앞부분 정보가 희석되는 문제가 생긴다 → Transformer가 이 한계를 해결</p>
