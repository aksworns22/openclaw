## LangGraph로 상태 그래프 만들기


<p>LangGraph는 노드(작업 단위)와 엣지(흐름)로 구성된 그래프를 정의해서 상태를 단계별로 변경해가는 실행 흐름을 만든다. 가장 단순한 형태로 카운터를 1씩 증가시키는 그래프를 만들어 동작 방식을 확인했다.</p>

<ul>
  <li><strong>상태(state)</strong>는 <code>TypedDict</code>로 정의한다. 그래프를 거치는 동안 노드들이 공유하고 갱신하는 데이터 구조다.</li>
  <li><strong>노드(node)</strong>는 현재 상태를 받아 갱신할 부분만 담은 딕셔너리를 반환하는 함수다.</li>
  <li><strong>엣지(edge)</strong>는 노드 간 실행 순서를 정의한다. <code>START</code>와 <code>END</code>는 그래프의 시작/끝을 나타내는 특수 노드다.</li>
  <li>그래프를 다 구성한 뒤 <code>compile()</code>로 실행 가능한 앱을 만들고, <code>invoke()</code>에 초기 상태를 넘겨 실행한다.</li>
</ul>

<div class="mermaid">
graph LR
  START --> increment["increment<br>(count += 1)"]
  increment --> END
</div>

<details>
<summary>예제 코드 보기</summary>
<pre><code>class CounterState(TypedDict):
    count: int


def increment(state: StateGraph) -> CounterState:
    new_count = state['count'] + 1
    return {"count": new_count}

graph = StateGraph(CounterState)
graph.add_node("increment", increment)
graph.add_edge(START, "increment")
graph.add_edge("increment", END)

app = graph.compile()
result = app.invoke({"count": 0})
</code></pre>
</details>

## 어텐션: 쿼리, 키, 값


<p>어텐션은 사람이 단어 사이의 관계를 고민하는 과정을 딥러닝 모델이 모방하도록 만든 연산이다. 사람은 단어를 그 자체로만 해석하지 않고 주변 단어로 맥락을 더해 이해하는데, 어텐션은 이 과정을 쿼리/키/값이라는 세 가지 벡터의 상호작용으로 구현한다.</p>

<ul>
  <li><strong>쿼리(query)</strong>: 검색창에 입력하는 검색어처럼, 지금 보고 있는 토큰이 "무엇과 관련 있는지" 묻는 질문 역할.</li>
  <li><strong>키(key)</strong>: 쿼리와의 관련도를 계산하기 위해 각 토큰(문서)이 가진 특징.</li>
  <li><strong>값(value)</strong>: 관련도가 계산된 후 실제로 가져오고 싶은 정보, 즉 검색 결과 자체.</li>
</ul>

<p>모든 토큰을 동등한 비중으로 평균 내는 방식으로는 "관련이 깊은 단어를 더 많이 반영한다"는 목표를 달성할 수 없다. 그래서 트랜스포머는 가중치 행렬 <code>W_q</code>, <code>W_k</code>, <code>W_v</code>를 학습시켜, 토큰 임베딩으로부터 쿼리·키·값을 만들고 토큰 사이의 관련도(가중치)를 계산하는 능력 자체를 모델이 학습하도록 한다.</p>

<img src="../raw/llm을 활용한 실전 AI 애플리케이션 개발/img/Pasted%20image%2020260621132147.png" alt="쿼리, 키, 값 생성 및 가중합 과정">

<p>예: "파리"라는 토큰의 쿼리(<code>q1</code>)가 문장 내 다른 토큰들의 키와 관련도(0, 0, 0.6, 0.2, 0.2)를 계산하고, 그 관련도로 각 토큰의 값을 가중합해 "파리"의 새로운 임베딩을 만든다. 결과적으로 정적인 단어 임베딩이 주변 맥락 정보를 반영한 임베딩으로 바뀐다.</p>

## 어텐션 직접 구현하기


<p>위 개념을 PyTorch로 직접 구현해 보면 다음 순서로 진행된다.</p>

<div class="mermaid">
graph LR
  text["입력 문장<br>(띄어쓰기 토큰화)"] --> ids["토큰 ID<br>(str2idx)"]
  ids --> emb["토큰 임베딩 + 위치 인코딩"]
  emb -->|W_q, W_k, W_v| qkv["쿼리/키/값"]
  qkv --> attn["스케일드 닷프로덕트 어텐션"]
  attn --> out["맥락이 반영된 임베딩"]
</div>

<ul>
  <li>띄어쓰기 단위로 토큰을 나누고 <code>str2idx</code>/<code>idx2str</code> 딕셔너리로 토큰 ↔ ID를 매핑한다.</li>
  <li><code>nn.Embedding</code>으로 토큰 임베딩과 위치 임베딩을 각각 만들고 더해서 입력 임베딩을 구성한다.</li>
  <li><code>nn.Linear</code> 세 개(<code>weight_q</code>, <code>weight_k</code>, <code>weight_v</code>)로 입력 임베딩을 쿼리/키/값으로 변환한다.</li>
  <li>스케일드 닷프로덕트 어텐션: <code>(쿼리 @ 키.transpose) / sqrt(head_dim)</code>으로 점수를 구하고 <code>softmax</code>로 가중치를 만든 뒤 값과 가중합한다.</li>
</ul>

<details>
<summary>구현 코드 보기</summary>
<pre><code># 토큰화 및 ID 매핑
input_text = "나는 최근 파리 여행을 다녀왔다"
input_text_list = input_text.split()
str2idx = {word: idx for idx, word in enumerate(input_text_list)}
input_ids = [str2idx[word] for word in input_text_list]

# 토큰 임베딩 + 위치 인코딩
embedding_dim = 16
max_position = 12
embed_layer = nn.Embedding(len(str2idx), embedding_dim)
position_embed_layer = nn.Embedding(max_position, embedding_dim)

position_ids = torch.arange(len(input_ids)).unsqueeze(0)
position_encodings = position_embed_layer(position_ids)
token_embeddings = embed_layer(torch.tensor(input_ids)).unsqueeze(0)
input_embeddings = token_embeddings + position_encodings  # (1, 5, 16)

# 쿼리, 키, 값 생성
head_dim = 16
weight_q = nn.Linear(embedding_dim, head_dim)
weight_k = nn.Linear(embedding_dim, head_dim)
weight_v = nn.Linear(embedding_dim, head_dim)

queries = weight_q(input_embeddings)
keys = weight_k(input_embeddings)
values = weight_v(input_embeddings)

# 스케일드 닷프로덕트 어텐션
def compute_attention(queries, keys, values, is_causal=False):
    dim_k = queries.size(-1)
    scores = queries @ keys.transpose(-2, -1) / sqrt(dim_k)
    weights = F.softmax(scores, dim=-1)
    return weights @ values

after_attention_embeddings = compute_attention(queries, keys, values)
# 원본 입력 형태: (1, 5, 16) -> 어텐션 적용 후 형태: (1, 5, 16)

# 재사용 가능한 어텐션 헤드 모듈
class AttentionHead(nn.Module):
    def __init__(self, token_embed_dim, head_dim, is_causal=False):
        super().__init__()
        self.is_causal = is_causal
        self.weight_q = nn.Linear(token_embed_dim, head_dim)
        self.weight_k = nn.Linear(token_embed_dim, head_dim)
        self.weight_v = nn.Linear(token_embed_dim, head_dim)

    def forward(self, queries, keys, values):
        return compute_attention(
            self.weight_q(queries),
            self.weight_k(keys),
            self.weight_v(values),
            is_causal=self.is_causal
        )
</code></pre>
</details>

<p>어텐션을 적용해도 텐서의 형태(<code>(1, 5, 16)</code>)는 그대로 유지된다 — 바뀌는 것은 각 토큰 임베딩이 담고 있는 정보(주변 맥락 반영 여부)다.</p>

## 허깅페이스 트랜스포머 라이브러리


<ul>
  <li>허깅페이스 트랜스포머는 다양한 트랜스포머 모델을 통일된 인터페이스로 사용할 수 있게 해주는 오픈소스 라이브러리다.</li>
  <li>허깅페이스는 모델을 <strong>바디(body)</strong>와 <strong>헤드(head)</strong>로 구분한다. 바디는 BERT 등 사전학습된 공통 인코더이고, 헤드는 그 위에 붙는 과제별 출력 층이다.</li>
  <li><code>AutoModel</code>은 모델의 바디만 불러오는 클래스다 — 헤드는 풀고자 하는 과제(텍스트 분류, 토큰 분류 등)에 맞게 별도로 구성한다.</li>
</ul>

<img src="../raw/llm을 활용한 실전 AI 애플리케이션 개발/img/Pasted%20image%2020260621135330.png" alt="바디와 헤드의 구분: 텍스트 분류 헤드 vs 토큰 분류 헤드">

<p>같은 BERT 바디(H1~H8 은닉 상태)를 공유하면서도, 위에 붙는 선형 층(헤드)을 바꾸는 것만으로 문장 전체에 대한 분류(텍스트 분류 헤드)와 각 토큰에 대한 분류(토큰 분류 헤드)라는 서로 다른 과제를 풀 수 있다.</p>
